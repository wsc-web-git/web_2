Summary for Students – Learn Logic in Simple Words

Here is a simple summary for students. In logic, we study statements that can be true or false. When we make a truth table, we look at the last column to decide the type of the statement pattern. If every value in the last column is True, we call it a tautology. A tautology is always true, no matter what values you put for p and q. It is like saying it will rain or it will not rain. You can never be wrong because one part must happen. If every value in the last column is False, we call it a contradiction. A contradiction is always false in every possible case. It is like saying you are sitting and standing at the same time. This can never happen in real life. If the last column has some True and some False mixed together, we call it a contingency. A contingency depends on the situation. It can be true sometimes and false other times. It is like saying you will find money tomorrow. You may find it or you may not.

Remember the short forms used in exams. Tautology is written as t and contradiction is written as c. Sometimes in old books, teachers use the word fallacy instead of contradiction, but the meaning is exactly the same. These three ideas are very useful not only in mathematics but also in computer programming and daily reasoning. In coding, if a programmer puts a tautology in a while loop condition, the loop becomes true forever and causes an infinite loop that freezes the program. If a programmer puts a contradiction in an if condition, that code will never run because the condition can never be true. That is why good programmers try to write contingencies so the program can make real choices. Practice making small truth tables every day. Start with simple patterns like p or not p and p and not p. Check the last column carefully. Soon you will be able to spot a tautology, contradiction, and contingency in just a few seconds during exams.
